📚 CLAT Prep Hub Architecture

Next.js 16 · Supabase · DeepSeek + Gemini AI · Custom cookie auth

📦 Tech Stack

  • • Frontend: Next.js 16 (App Router) + React 19 + TypeScript
  • • Styling: Tailwind CSS v4 + Lucide React icons
  • • Auth: Custom cookie-based (clat-at / clat-rt) + Supabase Auth
  • • Database: Supabase (PostgreSQL) with RLS
  • • AI: DeepSeek Chat (primary) + Gemini 2.5 Flash (fallback)
  • • Hosting: Vercel (Free tier, auto-deploy from GitHub)
  • • Version: Next.js 16.2.10, @supabase/ssr 0.12

🧠 AI Engine Features

  • • 5 parallel sub-agents for each CLAT section
  • • Per-section targets: English 22-26, CA 28-32, Legal 28-32, LR 22-26, Quant 10-14
  • • Dynamic batch sizing via maxQuestions param
  • • JSON validator + normalizer handles AI output quirks
  • • Deduplication against existing questions in DB
  • • Auto fallback: DeepSeek → Gemini if configured
  • • Rich per-section system prompts with CLAT 2027 syllabus

🔐 Auth & Routing

  • • Custom cookie auth: clat-at (access) + clat-rt (refresh)
  • • Edge middleware guards all /student and /admin routes
  • • SSR-safe — cookies survive across page loads
  • • Auto token refresh via refreshSession() fallback
  • • Role-based redirects: student ↔ admin dashboards
  • • Service role key for admin-only DB operations
  • • Session persistence via persistSessionToCookie()
